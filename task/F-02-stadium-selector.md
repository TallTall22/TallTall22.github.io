> **Generated by**: BDD Requirements Expert (Recursive BDD-Atomic Framework)
> **Feature**: F-02 — Allow users to select their starting MLB stadium (home base) for trip generation
> **Stack**: Vue 3 + TypeScript strict + Vuetify 3 + Pinia + Vite + Vitest

---

## Executive Summary

**What**: A searchable dropdown that lets users pick one of 30 MLB stadiums as their trip's departure point. Selection is persisted in `tripStore` as `homeStadiumId`. Each option displays the team logo (MLB CDN SVG) and team/stadium name.

**Why**: `homeStadiumId` is a required input for trip generation (F-03+). This is the first required user action after (or alongside) date range selection.

**Scope — IN**:
- Static JSON data file for all 30 MLB stadiums (2026 season)
- `tripStore` extension: add `homeStadiumId` ref + `setHomeStadium` action
- `stadiumService`: async loader with error handling
- `useStadiumSelector` composable: search/filter, selection, loading state
- `StadiumSelector.vue`: Vuetify `v-autocomplete` with logo + name display
- Integration into `App.vue` alongside `DateRangePicker`
- Unit tests for composable

**Scope — OUT**:
- Map display of selected stadium (F-03)
- Stadium detail page / modal
- Dynamic data (MLB API calls)
- User geolocation auto-detection

## Domain Analysis & Recursive Validation

### Given-When-Then Scenarios

**Scenario 1: Primary Flow — User Selects Home Stadium**

- **Given**: App is mounted, stadiums.json has loaded successfully (30 items), `homeStadiumId` in store is `null`
- **When**: User opens the `StadiumSelector` dropdown and types "New York" or scrolls to select "New York Yankees"
- **Then**:
  - Dropdown filters list to teams matching the search string (teamName, stadiumName, city all searched)
  - User sees team logo SVG + `"New York Yankees — Yankee Stadium"` in each option
  - On selection, `tripStore.homeStadiumId` is set to `'NYY'`
  - Dropdown displays the selected team's name + logo in the trigger
  - No emit needed — store update is the side effect

**Scenario 2: Search Filters Across Multiple Fields**

- **Given**: Dropdown is open, 30 stadiums loaded
- **When**: User types "Chicago"
- **Then**: Two results appear — Chicago Cubs (Wrigley Field) and Chicago White Sox (Guaranteed Rate Field)

**Scenario 3: Search Yields No Results**

- **Given**: Dropdown is open, 30 stadiums loaded
- **When**: User types "zzz" (no match)
- **Then**: Vuetify `v-autocomplete` shows its built-in no-data slot with message "沒有符合的球隊"

**Scenario 4: Data Loading Failure**

- **Given**: App mounts but `stadiums.json` fetch/import fails (corrupt file, network error in future)
- **When**: `stadiumService.loadStadiums()` rejects
- **Then**:
  - `isLoading` → `false`, `loadError` → non-null error message
  - `StadiumSelector` renders a disabled input with error message "無法載入球隊資料，請重新整理"
  - No crash, no unhandled rejection

**Scenario 5: User Clears Selection**

- **Given**: `homeStadiumId` is `'NYY'`
- **When**: User clicks the clear (×) button on the autocomplete
- **Then**: `tripStore.homeStadiumId` is set back to `null`

**Scenario 6: Component Remount / Persistence**

- **Given**: User has selected "Boston Red Sox", navigates away, returns
- **When**: `StadiumSelector` remounts
- **Then**: Dropdown pre-populates with "Boston Red Sox" from `tripStore.homeStadiumId` (Pinia state persists for SPA lifetime)

### Technical Feasibility Assessment

| Concern | Finding | Mitigation |
|---|---|---|
| **Async Loading** | `import('@/assets/data/stadiums.json')` is a static JSON dynamic import — Vite bundles it, low failure risk, but must handle gracefully | Wrap in try/catch in service; expose `loadError` ref in composable |
| **Race Condition** | Only one load call at mount; no concurrent async calls | Guard with `isLoading` flag; call once in `onMounted` |
| **Prop Drilling** | `homeStadiumId` goes directly into `tripStore` — zero prop drilling | Composable reads/writes store directly via `storeToRefs` |
| **Type Safety** | `Stadium` interface already defined; need `StadiumSelectorOption` for v-autocomplete item shape | Define explicit `StadiumSelectorOption` type; never use `any` for item-slot props |
| **v-autocomplete item-title** | Vuetify needs a `string` field for display + filtering | Use `itemTitle` function or set `:item-title` to a computed label; define `StadiumSelectorOption.label` |
| **Logo 404 Risk** | `mlbstatic.com` CDN is official MLB — stable. But must handle SVG load failures in `<img>` | Add `@error` handler on `<img>` to show fallback emoji ⚾ |
| **Performance** | 30 items — trivial. No virtualization needed | No memoization required; `v-autocomplete` handles internally |
| **Memory Leak** | Watcher in composable for store sync | No watchers needed — `storeToRefs` provides reactive refs directly |
| **TypeScript `any`** | Vuetify slot props (e.g., `item` in `#item` slot) default to `any` | Cast with explicit `as StadiumSelectorOption` in template; use typed slot prop interfaces |
| **State Persistence** | SPA lifetime only (Pinia in-memory) — acceptable for F-02 | No localStorage needed at this stage |

  ```typescript
  // src/services/stadiumService.ts
  import type { Stadium, StadiumLoadResult } from '@/types/models';

  /**
   * Loads stadium data from the static JSON asset.
   * Uses dynamic import so Vite can tree-shake and chunk correctly.
   * Returns a StadiumLoadResult — never throws.
   */
  export async function loadStadiums(): Promise<StadiumLoadResult> {
    try {
      const module = await import('@/assets/data/stadiums.json');
      const raw: unknown = module.default;

      if (!Array.isArray(raw) || raw.length === 0) {
        return { stadiums: [], error: 'EMPTY_DATA' };
      }

      // Cast is safe: JSON shape is validated by Stadium interface at build time
      return { stadiums: raw as Stadium[], error: null };
    } catch {
      return { stadiums: [], error: 'FETCH_FAILED' };
    }
  }
  ```

  **TypeScript surface**:
  - Return type: `Promise<StadiumLoadResult>`
  - Zero `any` — the `unknown` cast is intentional and immediately narrowed

  **Error handling**: All branches return `StadiumLoadResult`; never throws to caller.

  **Testing surface**: Export `loadStadiums` as named export for easy mock injection.

  > **Note**: The return interface above is illustrative — write it concisely in the actual file (use explicit `Ref<T>` / `ComputedRef<T>` imports from `vue`). Full implementation:

  ```typescript
  // src/composables/useStadiumSelector.ts
  import { ref, computed, onMounted } from 'vue';
  import type { Ref, ComputedRef } from 'vue';
  import { storeToRefs } from 'pinia';
  import { useTripStore } from '@/stores/tripStore';
  import { loadStadiums } from '@/services/stadiumService';
  import type { Stadium, StadiumLoadErrorCode } from '@/types/models';
  import type { StadiumSelectorOption } from '@/types/components';

  export function useStadiumSelector() {
    const store = useTripStore();
    const { homeStadiumId } = storeToRefs(store);

    const stadiums  = ref<Stadium[]>([]);
    const isLoading = ref<boolean>(false);
    const loadError = ref<StadiumLoadErrorCode | null>(null);

    /** Map each stadium to an autocomplete option with pre-built searchable label */
    const options = computed<StadiumSelectorOption[]>(() =>
      stadiums.value.map((s) => ({
        label: `${s.teamName} — ${s.stadiumName}`,
        stadium: s,
      }))
    );

    /** Sync the currently-selected option from the store's homeStadiumId */
    const selectedOption = computed<StadiumSelectorOption | null>(() => {
      if (!homeStadiumId.value) return null;
      return options.value.find((o) => o.stadium.id === homeStadiumId.value) ?? null;
    });

    function onSelect(option: StadiumSelectorOption | null): void {
      store.setHomeStadium(option?.stadium.id ?? null);
    }

    function onClear(): void {
      store.setHomeStadium(null);
    }

    onMounted(async () => {
      isLoading.value = true;
      const result = await loadStadiums();
      stadiums.value = result.stadiums;
      loadError.value = result.error;
      isLoading.value = false;
    });

    return {
      stadiums,
      options,
      homeStadiumId,   // expose for v-model binding on v-autocomplete
      selectedOption,
      isLoading,
      loadError,
      onSelect,
      onClear,
    };
  }
  ```

  **State flow**:
  - Data enters via `loadStadiums()` on `onMounted`
  - Selection exits via `store.setHomeStadium()` — no emits, no prop drilling
  - `homeStadiumId` from `storeToRefs` keeps template in sync with store reactively

  **Error handling**:
  - `loadError` is non-null when service fails; component reads it to show disabled state
